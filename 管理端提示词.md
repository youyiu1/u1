# 同城生活管理端系统提示词

你正在为“同城生活社区平台”开发管理端，请严格遵守以下要求：

## 目标
- 开发一个独立的后台管理系统，用于平台运营管理。
- 重点覆盖：用户、动态、闲置商品、服务、订单、通知、分类。
- 风格简洁、清晰、专业、易维护。

## 项目技术栈
- 前端：React 19、Vite、TypeScript、Tailwind CSS v4、React Router v7
- 后端：Spring Boot 3.2、MyBatis-Plus、MySQL、Redis
- 接口统一返回：`{ success, message, data, total }`
- 认证方式：JWT + Redis，请求需携带 `Authorization: Bearer token`

## 管理端功能
1. 登录鉴权
   - 管理员登录
   - token 持久化
   - 未登录自动跳转登录页

2. 仪表盘
   - 用户数、动态数、商品数、服务数、订单数
   - 最新动态、最新订单、待处理事项

3. 用户管理
   - 列表、搜索、详情
   - 昵称、邮箱、头像、认证状态、关注数据
   - 启用、禁用、删除

4. 动态管理
   - 列表、筛选、搜索
   - 内容、图片、作者、点赞、评论、时间
   - 删除违规内容、管理评论

5. 闲置商品管理
   - 列表、筛选、搜索
   - 标题、价格、成色、位置、卖家、图片
   - 下架、删除

6. 服务管理
   - 列表、筛选、搜索
   - 标题、价格、评分、评价数、服务者信息
   - 审核、下架、删除

7. 订单管理
   - 列表、状态筛选、详情
   - 买家、卖家、服务、价格、预约时间、状态
   - 异常订单处理

8. 通知管理
   - 系统通知、用户通知
   - 支持发送通知

9. 分类管理
   - 动态、商品、服务分类
   - 新增、编辑、删除

## 页面与交互
- 使用“左侧菜单 + 顶部栏 + 内容区”布局。
- 表格支持分页、搜索、筛选、操作。
- 重要操作必须二次确认。
- 必须有加载、空状态、错误状态反馈。
- 优先保证桌面端体验，移动端基础适配即可。

## 开发约束
- 优先复用现有代码和依赖。
- API 请求统一封装，复用现有 token 逻辑。
- 新增接口必须在 Controller 方法上方添加中文注释。
- Long 类型 ID 返回前端时要避免 JS 精度丢失。
- 新增表或字段时同步更新 `backend-java-reference/sql/init.sql`。
- 前端改动后运行 `npm run lint`。
- 后端改动后运行 `mvn -q -DskipTests compile`。

## 推荐目录
- 前端：`frontend/src/pages/admin/`、`frontend/src/components/admin/`、`frontend/src/services/adminApi.ts`
- 后端：`backend-java-reference/src/main/java/com/neighborhood/app/controller/AdminController.java`

## 输出要求
- 按模块逐步实现。
- 优先完成：基础布局、登录鉴权、仪表盘、用户管理。
- 后续扩展：动态、商品、服务、订单、通知、分类。
- 保持代码简洁、清晰、可维护。
